Core Concepts

What is Artificial Intelligence?

AI is the simulation of human intelligence processes by computer systems. It encompasses three main categories:

  • Narrow AI (weak AI): Systems designed for specific tasks (all current AI systems)
  • General AI (strong AI): Hypothetical systems with human-level intelligence across domains
  • Super AI (ASI): Theoretical AI surpassing human intelligence (not yet realized)

Machine Learning vs. Deep Learning vs. AI

Understanding the relationship between these terms:

AI (Broadest)
→ Any technique that enables computers to mimic human behavior
Machine Learning (Subset of AI)
→ Systems that learn and improve from experience without explicit programming
Deep Learning (Subset of ML)
→ Neural networks with multiple layers that learn from large amounts of data

Types of Machine Learning

  • Supervised Learning: Learning from labeled data (email spam detection)
  • Unsupervised Learning: Finding patterns in unlabeled data (customer segmentation)
  • Reinforcement Learning: Learning through rewards and penalties (game AI)

The ML Pipeline: From Data to Predictions

  1. Data Collection: Gather relevant examples (emails, transactions, images, etc.)
  2. Data Preparation: Clean, label, and format data for learning
  3. Feature Engineering: Identify which characteristics matter most
  4. Model Training: Expose the model to patterns in data
  5. Model Evaluation: Test accuracy on new, unseen examples
  6. Deployment: Put the model into production
  7. Monitoring: Track performance over time

📖 Practical Example: Email Spam Detection

The Problem: Gmail receives 200+ million spam emails per day. Humans can't review each one.

The AI Solution:

  1. Train: Feed the model millions of labeled emails (spam/not spam)
  2. Learn: Model identifies patterns (sender reputation, keywords, links)
  3. Predict: New email is analyzed and classified in milliseconds
  4. Improve: User feedback retrains the model

Key Takeaway: AI excels at pattern recognition in large datasets—it won't "understand" emails like humans, but it spots patterns humans would miss.

🎓 Key Insights for Product Owners

  • AI is NOT magical—it's pattern matching at scale
  • AI needs data, lots of it. Quality matters more than quantity
  • AI works best for well-defined, specific problems (not vague ones)
  • AI cannot replace human judgment; it augments it
  • The model is only as good as the data it was trained on
  • Bias in training data becomes bias in predictions

⚙️ Common ML Models You Should Know

Linear Regression

Predicts continuous values (house prices, revenue forecasts)

Logistic Regression

Predicts binary outcomes (spam/not spam, click/no click)

Decision Trees & Random Forests

Like flowcharts—easy to interpret (loan approval, medical diagnosis)

Neural Networks

Complex pattern recognition (image recognition, NLP)

⚡ The AI Feasibility Challenge

Before building AI into your product, ask yourself:

  • Is there enough data? (typically 1000+ examples minimum)
  • Can you define success? (metrics, not vague goals)
  • Is this repeatability? (same input = same pattern each time)
  • Will it improve user value? (not just cool tech)
  • Can you afford to be wrong? (what's the cost of errors?)

If you can't answer "yes" to most of these, first-generation AI might not be your solution.

✏️ Your Exercise This Week

Identify AI in Your Product

Think of 3 features in products you use daily that could be powered by AI or ML. For each one:

  1. What is the problem it solves? (what user pain does it address?)
  2. What type of data would it need? (and do we have access to it?)
  3. How would you measure success? (metrics, not vibes)
  4. Could traditional algorithms work instead? (is ML really necessary?)

💡 Tip: Write your answers down. You'll reference this in Phase 2 when building strategy.
