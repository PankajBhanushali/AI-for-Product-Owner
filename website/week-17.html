The A/B Testing Framework

The Scientific Method for AI

Three Steps to Rigorous Testing:

Step 1: Hypothesis & Metrics

What do we expect will happen? What metrics prove it?

Step 2: Test Design

Sample size, duration, how to segment users

Step 3: Analysis & Decision

Is difference significant? Ship or iterate?

Test Design Essentials

Key decisions before launching a test:

  • Primary metric: The one metric that matters most. Not more than 1-2.
  • Sample size: How many users must see each experience? Use power analysis (typically 5k-50k per group)
  • Duration: How long to run? Balance: long enough for stability, short enough to decide fast (typically 1-2 weeks)
  • Minimum detectable effect: What's the smallest improvement worth shipping? If you need 50% lift to be viable, don't bother testing 10% improvements
  • Segmentation: Test on all users or subset? New users only? High-value users?

Statistical Power & Significance

Understanding test validity:

Statistical Significance (p-value < 0.05)

Only 5% chance the difference is due to random variation. Gold standard for shipping.

Statistical Power (typically 80%)

80% probability we'll detect the improvement if it truly exists. Tells you sample size needed.

Confidence Interval

Range estimation: "The improvement is between 5-15%, 95% confident"

Common Testing Pitfalls

Avoid these mistakes:

  • Peeking at results: Checking winner mid-test inflates true false-positive rate. Set duration upfront.
  • Multiple comparisons: Testing 100 metrics? By chance 5 will be "significant". Stick to 1-2 primary metrics.
  • Sample size too small: Can miss real improvements. Use power analysis.
  • Externalities: Major news, holiday, competitor launch—can confound results. Document context.
  • Novelty effect: New feature is cool at first, then engagement drops. Test long enough (2+ weeks)

✏️ Your Exercise This Week

Design Your First AI A/B Test

  1. Define hypothesis: "AI feature will improve [metric] by [X]%"
  2. Choose primary metric and success threshold
  3. Estimate sample size (use online calculator)
  4. Determine test duration
  5. Define exclusions (new users? non-English?)
  6. Create testing checklist and rollback plan
