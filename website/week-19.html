🚀 Week 19: Capstone Project - Build Your AI Feature

Applying everything to build a real AI product

⏱️ Duration: 1 week
📚 Content: 10+ hours (hands-on building)
🎯 Level: Advanced

🎯 Your Capstone Challenge

Define the Problem

Choose real problem: your company or public data

Solution Discovery

Opportunity mapping and strategy canvas

Build & Ship

Create prototype and go through gate approvals

Measure Impact

Define metrics and validation approach

The Capstone Project Framework

Phase 1: Problem Definition (Days 1-2)

What problem will you solve?

  • Option A: Real problem from your company (ideal)
  • Option B: Public dataset (Kaggle, UCI ML Repository)
  • Option C: Synthetic scenario (e.g., "Build a customer churn prediction model")

Deliverable: 1-page problem statement: What's the opportunity? Who benefits? Why does it matter?

Phase 2: Solution Design (Days 2-3)

How will AI solve it?

  • Opportunity Mapping: Customer journey, pain points, where does AI fit?
  • Strategy Canvas: Value prop, target customer, business model, competitive advantage
  • Use Case Definition: Input, output, success criteria, edge cases
  • Tech Stack: API (OpenAI, Hugging Face) or build custom model?

Deliverable: Strategy canvas + 1-page design doc. Include wireframe/mockup of how user interacts with AI.

Phase 3: Building (Days 4-5)

Create a working prototype:

  • Data: Gather and clean dataset (or use public data)
  • Model: Train or integrate API. Aim for simple > complex.
  • Integration: Build simple UI (web app, API endpoint, notebook)
  • Testing: Test on sample cases. Document edge case failures.

Deliverable: Working prototype (GitHub repo with code + README + instructions to run)

Phase 4: Risk & Ethics (Day 6)

Address the governance questions:

  • Risk Register: What could go wrong? Mitigation strategies?
  • Fairness Audit: Does model work equally well for all groups?
  • Compliance Checklist: GDPR? Data privacy? Industry regulations?
  • Monitoring Plan: What metrics will you track post-launch?

Deliverable: Risk register (5-10 risks, mitigation plans) + fairness assessment

Phase 5: Go-to-Market & Metrics (Day 7)

How will you ship and measure success?

  • Launch Plan: Shadow mode → beta → general availability
  • Success Metrics: Primary + secondary metrics, targets
  • A/B Test Design: How will you measure if it works?
  • Cost Analysis: Estimate monthly costs and ROI

✏️ Capstone Deliverables

What to Deliver (Start of Week 20)

  • ✅ Problem Statement: 1 page explaining the problem and opportunity
  • ✅ Strategy Canvas: Visual showing product strategy + value prop
  • ✅ Design Doc / Wireframes: How users interact with the AI feature
  • ✅ GitHub Repo: Code, data, trained model, README with instructions
  • ✅ Risk Register: 5-10 identified risks + mitigation strategies
  • ✅ Fairness Assessment: Testing for bias across demographic groups
  • ✅ Metrics Framework: ML metrics + business metrics + success targets
  • ✅ Launch Plan: Rollout strategy (shadow → beta → public) + exit criteria
  • ✅ Cost Model: Estimated monthly costs and ROI projection
  • ✅ Executive Summary: 2-page overview of entire project
