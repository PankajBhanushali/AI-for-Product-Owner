Planning AI Features

Writing AI Feature PRDs

Standard PRD structure, with AI-specific sections:

1. Overview & Goals

What AI capability? What problem does it solve? Why now?

2. Model Requirements

Build vs Buy: Use existing model (GPT-4, Claude) or build custom?

Performance Requirements: Accuracy target, latency (ms), cost per request

3. Data Requirements

Data Sources: Where does training/inference data come from?

Data Volume: How many examples needed? (min: 100s, good: 1000s+)

Privacy/Security: PII handling? Data residency? Compliance?

4. Infrastructure

Deployment: On-prem, cloud, edge? Real-time or batch?

Monitoring: How do we track performance drift over time?

5. Risks & Mitigation

Hallucinations? Bias? Privacy? Cost overruns?

For each: severity, likelihood, and mitigation plan

Build vs. Buy Decision

Build Custom Model If:

  • You have unique competitive data (that trains better models)
  • Off-the-shelf models don't meet your accuracy needs
  • Cost of API calls exceeds training investment
  • Privacy requires keeping data on-premises

Use Existing Models (GPT-4, Claude, etc) If:

  • Off-the-shelf works well enough
  • You don't want to maintain ML infrastructure
  • Speed to market matters more than perfection
  • Lower volume (API costs are manageable)

General Rule: Start with existing models. Only custom if it becomes a bottleneck.

Timeline & Effort Estimation

Typical AI project phases:

Phase 1: Exploration (2-4 weeks)

Gather data, test models, establish baselines

Phase 2: Development (4-12 weeks)

Build pipeline, train/fine-tune model, test thoroughly

Phase 3: Integration (2-4 weeks)

Connect to product, human review layer, monitoring

Phase 4: Launch & Monitor (Ongoing)

Gradual rollout, monitoring, retraining as needed

⚠️ Don't underestimate: Data work (50% of effort), monitoring/maintenance, retraining.

✏️ Your Exercise This Week

Write an AI Feature PRD

  1. Create a detailed PRD for one AI feature
  2. Include all 5 sections: overview, model requirements, data, infrastructure, risks
  3. Make Build vs Buy decision with justification
  4. Create realistic timeline with phases
  5. Identify team roles and hiring needs
