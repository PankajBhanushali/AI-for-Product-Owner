Data Strategy Fundamentals

Data is the New Oil

Good AI requires good data. The formula:

  • Volume: For supervised learning: 100s minimum, 1000s+ ideal
  • Quality: Accurate labels. No mislabeling (worst!
  • Diversity: Represents all use cases (not biased toward one segment)
  • Freshness: Data reflects current conditions (stale data = bad predictions)

Rule: Data quality matters more than quantity. 1000 perfect labels > 1M mislabeled ones.

The Data Workflow

1. Collection

Where does data come from? Product logs, user interactions, manual uploads

2. Storage

Data warehouse, data lake, or database

3. Cleaning

Remove duplicates, fix errors, handle missing values

4. Labeling

Add ground truth (what the right answer is)

5. Feature Engineering

Transform raw data into useful signals for models

6. Validation

Split into train/test sets, check for bias

7. Documentation

Datasheets explaining data, sources, limitations

Data Labeling Strategies

Labeling is often the bottleneck. Three approaches:

1. Manual Labeling

Humans label every example

Cost: $0.10-1 per label. Quality: High. Speed: Slow for large datasets

2. Weak Supervision

Use heuristics or existing systems to auto-label

Cost: Low. Quality: Noisy. Speed: Fast

3. Active Learning

Model identifies most uncertain examples for humans to label

Cost: Medium. Quality: High. Value/label: High

✏️ Your Exercise This Week

Build Your Data Strategy Document

  1. Identify all data sources for your AI features
  2. Estimate data volume available today
  3. Define data quality metrics and validation rules
  4. Choose labeling strategy (manual/weak/active)
  5. Create data governance policy (privacy, access, retention)
  6. Document data lineage and ownership
