# Contributing to AI for Product Owner

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## Contribution Workflow

All contributions to this project must follow our **Pull Request (PR) Strategy**. Direct commits to the `main` branch are not permitted.

### Step 1: Fork the Repository
```bash
# Fork the repository on GitHub
# Then clone your fork locally
git clone https://github.com/YOUR-USERNAME/AI-for-Product-Owner.git
cd AI-for-Product-Owner
```

### Step 2: Create a Feature Branch
```bash
# Create a new branch for your changes
git checkout -b feature/your-feature-name
# Examples:
# feature/add-week-21-content
# fix/broken-links
# docs/improve-readme
```

### Step 3: Make Your Changes
- Follow the existing code style and formatting
- Keep commits atomic and descriptive
- Add meaningful commit messages

### Step 4: Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### Step 5: Create a Pull Request
1. Go to the original repository on GitHub
2. Click "New Pull Request"
3. Select your fork and feature branch
4. Fill in the PR template with:
   - Clear description of changes
   - Why this change is needed
   - Any related issues
5. Submit the PR for review

### Step 6: Code Review & Approval
- The repository maintainer will review your PR
- Address any requested changes
- Once approved, the PR will be merged into `main`

## PR Requirements

Your pull request should:
- ✅ Have a clear, descriptive title
- ✅ Include a detailed description of changes
- ✅ Reference any related issues (e.g., "Fixes #123")
- ✅ Follow the project's code style
- ✅ Include relevant tests or examples if applicable

## Branch Protection Rules

⚠️ **Important**: The `main` branch is protected with the following rules:
- ✅ Pull request reviews **required** before merging
- ✅ At least **1 approval** needed from repository maintainer
- ✅ Conversations must be resolved before merging
- ✅ Branches must be up-to-date before merging

**This means:**
1. You cannot push directly to `main`
2. All changes must come through approved pull requests
3. The maintainer must review and approve your PR

## Types of Contributions Welcome

- **New Content**: Additional weeks, phases, or curriculum modules
- **Improvements**: Better wording, clearer explanations, additional examples
- **Labs & Templates**: New templates, tools, or resources
- **Bug Fixes**: Fix broken links, typos, or errors
- **Documentation**: Update README, guides, and contribution docs
- **Website Improvements**: UI/UX enhancements, styling fixes

## Code Style

- Use clear, descriptive variable and file names
- Follow existing formatting patterns
- Keep content consistent with the rest of the project
- Test your changes before submitting
