📈 Week 18: AI Scaling & Cost Optimization

Managing AI costs and scaling operations

⏱️ Duration: 1 week
📚 Content: 5-6 hours
🎯 Level: Intermediate

🎯 What You'll Learn

Cost Drivers

API calls, compute, storage, labor

Cost Optimization

Caching, batching, model distillation

Infrastructure Scaling

Auto-scaling, load balancing, regionalization

ROI Calculation

Cost vs benefit analysis for AI features

AI Cost Architecture

The Cost Equation

Total AI Cost = API Costs + Compute + Storage + Labor

API Costs (Third-Party Models)

If using OpenAI, Anthropic, etc: $/token or $/request. Scales with volume.

Compute (GPUs, TPUs)

If hosting own model: cloud GPU $/hour or on-premise hardware cost. A100 ~$1.50/hour on GCP.

Storage (Vector DB, Training Data)

Vector embeddings, model checkpoints, training data. Pinecone ~$0.40/1M vectors.

Labor (Engineering, Data, Ops)

Often biggest cost: salaries. 1 ML engineer = $150k-$300k/year.

Cost Optimization Strategies

How to reduce AI costs without sacrificing quality:

  • Caching: Cache model outputs. If 100 users search for "Macbook", show cached results. Saves 99% of requests.
  • Batching: Group inference requests. 100 sequential requests = 1 batch request. Reduces latency, reduces cost.
  • Model distillation: Train smaller, faster model to mimic large one. Inference 10x faster, 90% accuracy retained.
  • Quantization: Compress model weights (FP32 → INT8). Smaller, faster, less memory.
  • Fallback models: Use fast, cheap model for 95% of cases. Reserve expensive model for edge cases.
  • Reserved capacity: Buy reserved GPUs on cloud. 70% cheaper than on-demand if you can commit upfront.

Scaling for Millions of Users

Infrastructure patterns that work at scale:

1. Load Balancing

Distribute requests across multiple GPU servers. If one dies, traffic reroutes.

2. Auto-scaling

Spin up more GPUs when latency exceeds threshold. Spin down in off-peak hours.

3. Multi-region Deployment

Deploy same model in US-East, US-West, EU. Users get lower latency.

4. Asynchronous Processing

Queue heavy requests. Process overnight. Return results via email/webhook.

ROI & Unit Economics

How to know if your AI feature is worth the cost:

  • Cost per inference: Total cost / number of inferences. If $0.001/prediction and you make 1M predictions/month = $1,000/month
  • Benefit per inference: How much revenue/value per prediction? If 10% conversions at $100 order value = $10 benefit per prediction
  • Payback period: How long until AI investment pays for itself? If costs $50k to build and saves $5k/month = 10 months payback
  • Gross margin impact: AI costs should be < 10% of revenue generated. If AI costs $10k/month and generates $100k/month in value, you're good.

✏️ Your Exercise This Week

Build Your AI Cost Model

  1. Estimate monthly API costs ($/request × monthly requests)
  2. Estimate infrastructure costs (GPU hours, storage, CDN)
  3. Estimate labor costs (engineer hours × rate)
  4. Calculate cost per inference
  5. Identify 3 cost optimization opportunities
  6. Build ROI calculator: cost vs projected revenue/benefit
