    Prompt Engineering Mastery

    The Prompt Formula

    A good prompt has these elements (in order):

    1. Role/Context

    "You are a customer support specialist for a SaaS platform..."

    2. Task Description

    "Your job is to respond to customer support emails..."

    3. Context/Background

    "Our product is X, our customers are Y, common issues are Z..."

    4. Examples (Few-Shot)

    1-5 examples of input/output showing expected style

    5. Constraints

    Word limits, tone, format, what to avoid

    6. The Actual Request

    "Here's the customer email. Please respond:"

    Advanced Technique: Chain-of-Thought

    Problem: LLMs make mistakes on complex reasoning

    Solution: Ask it to show its reasoning step-by-step

    ❌ Without Chain-of-Thought:

    Prompt: "Is this customer at churn risk? Yes or no."

    Output: "Yes" ← You don't know why!

    ✅ With Chain-of-Thought:

    Prompt: "Is this customer at churn risk? Think through this step-by-step: (1) What is their usage trend? (2) Have they opened support tickets? (3) Are they in a high-value segment? Then give your conclusion."

    Output: "(1) Usage down 40% this month. (2) Yes, 3 tickets about performance. (3) Yes, enterprise customer. Conclusion: High churn risk."

    Testing & Iteration

    The Prompt Engineering Cycle:

    1. Write a prompt: Start simple, iterate
    2. Test on samples: 10-20 test cases covering edge cases
    3. Grade outputs: Good/neutral/bad. What % are good?
    4. Analyze failures: Why did it fail? Common patterns?
    5. Refine prompt: Add examples, clarify instructions, adjust tone
    6. Repeat: Run 10+ iterations targeting 90%+ success

    Tool Tip: Use spreadsheets or tools like Promptfoo to track iterations and scores.

    Common Prompt Patterns

    Classification

    Categorizing inputs (e.g., ticket type, sentiment)

    Tip: List exact categories. Use examples of each.

    Extraction

    Pulling structured data from text

    Tip: Specify exact JSON format wanted. Use examples.

    Generation

    Creating new content (emails, summaries, code)

    Tip: Show style examples. Set tone & length constraints.

    Reasoning

    Complex multi-step analysis

    Tip: Use chain-of-thought. Show reasoning examples.

    ✏️ Your Exercise This Week

    Write & Test Production Prompts

    1. Identify 3 AI capabilities you need from Week 7-9
    2. For each, write 2 prompt versions (simple vs advanced)
    3. Test both on 10-15 sample inputs from your product
    4. Grade outputs (good/neutral/bad)
    5. Document the best-performing prompts
    6. Create a "Prompt Library" document for your team
