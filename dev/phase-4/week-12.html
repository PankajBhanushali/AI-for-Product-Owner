The ML Lifecycle

Six Stages of Model Development

Stage 1: Problem Definition

What are we trying to predict/classify? Success criteria?

Stage 2: Experimentation

Notebooks, try different models, iterate quickly

Stage 3: Testing & Validation

Evaluate on holdout test set, check for bias, stress-test

Stage 4: Deployment

Shadow mode → canary rollout → full rollout

Stage 5: Monitoring

Track performance in production, watch for degradation

Stage 6: Maintenance & Retraining

Update model regularly or when it drifts

Model Versioning & Registry

Why version models? Same as code—need to track what changed, revert if needed, compare performance

Model Registry should track:

  • Model ID, version, date created
  • Training data version and hyperparameters
  • Performance metrics on test set
  • Fairness checks (bias audit results)
  • Status (in development, in production, archived)
  • Who trained it and who approved it

Tools: MLflow, Weights & Biases, or custom registry

Monitoring & Alerting

What to monitor:

1. Model Performance

Accuracy, precision, recall. Alert if drops below threshold

2. Data Drift

Input data distribution changed. Model may be out of date.

3. Prediction Drift

Model predictions changing unexpectedly. May indicate issues.

4. Fairness

Disparate impact across demographic groups

5. System Metrics

Latency, throughput, error rates, cost

✏️ Your Exercise This Week

Design Your ML Lifecycle

  1. Document the 6 stages for your AI model
  2. Define success criteria for each stage
  3. Create a versioning scheme (how you'll track models)
  4. List all metrics to monitor post-launch
  5. Set alert thresholds (when should we retrain?)
  6. Document retraining schedule/triggers
